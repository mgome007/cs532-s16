	<p>A <a href="https://en.wikipedia.org/wiki/Red_team" target="_blank">Red Team</a> exercise is lotsa fun not only because you have a more realistic engagement due to the broader scope, but also because you can encounter situations which you normally wouldn&#8217;t on a regular narrow scoped penetration test. I&#8217;m going to focus on <a href="http://www.chiark.greenend.org.uk/~sgtatham/putty/download.html" target="_blank">pageant</a> which Slurpgeit recently encountered during one of these red team exercises which peeked my interest.</p>
<p>Apparantly he got access to a machine on which the user used pageant to manage ssh keys and authenticate to servers without having to type his key password every single time he connects. This of course raises the following interesting (or silly) question:</p>
<p><em>Why does the user only have to type his ssh key in once?</em></p>

<p><em>The pageant process keeps the decrypted key in memory so that you can use them without having to type the decryption password every time you want to use the key.</em></p>
<p>From an attackers perspective it of course begs the question if you can steal these unencrypted keys? Assuming you are able to make a memory dump of the running process you should be able to get these decrypted ssh keys. During this blog post I&#8217;ll be focusing on how you could achieve this and the pitfalls I encountered when approaching this.</p>

This morning it hit me, this is something we can actually accomplish with password hashes. We can rotate them continuously, without the user even noticing and hopefully make it harder for attackers to crack the hash. Which is the main goal of this &#8220;idea/solution&#8221; making it harder or hopefully impossible for attacker to crack the hash when they obtain it. Here is a PoC preview output, screenshot OTP and usage OTP don&#8217;t match because they were obtained at different times:</p>

	<small>
		Posted: December 10, 2014 in <a href="http://diablohorn.com/category/general/" rel="category tag">general</a>		<br />
		Tags: <a href="http://diablohorn.com/tag/hiberfil-sys/" rel="tag">hiberfil.sys</a>, <a href="http://diablohorn.com/tag/hibernation/" rel="tag">hibernation</a>, <a href="http://diablohorn.com/tag/python/" rel="tag">python</a>, <a href="http://diablohorn.com/tag/slack-space/" rel="tag">slack space</a>, <a href="http://diablohorn.com/tag/volatility/" rel="tag">volatility</a>	</small>

<div class="postcomments"><a href="http://diablohorn.com/2014/12/10/parsing-the-hiberfil-sys-searching-for-slack-space/#comments">4</a></div>
<div class="entry">
	<p>Implementing functionality that is already available in an available tool is something that has always taught me a lot, thus I keep on doing it when I encounter something I want to fully understand. In this case it concerns the &#8216;hiberfil.sys&#8217; file on Windows. As usual I first stumbled upon the issue and started writing scripts to later find out someone had written a nice article about it, which you can read <a title="Hibernation Slack: Unallocated Data from the Deep Past" href="http://digital-forensics.sans.org/blog/2014/07/01/hibernation-slack-unallocated-data-from-the-deep-past" target="_blank">here</a> (1). For the sake of completeness I&#8217;m going to repeat some of the information in that article and hopefully expand upon it, I mean it&#8217;d be nice if I could use this entry as a reference page in the future for when I stumble again upon hibernation files. Our goal for today is going to be to answer the following question:</p>
<blockquote><p>What&#8217;s a hiberfil.sys file, does it have slack space and if so how do we find and analyze it?</p></blockquote>
<p>To answer that question will hopefully be answered in the following paragraphs; we are going to look at the hibernation process, hibernation file, it&#8217;s file format structure, how to interpret it and finally analyze the found slack space. As usual you can skip the post and go directly to the <a title="find hibernation slack space" href="https://github.com/DiabloHorn/find_hib_slack" target="_blank">code</a>.</p>
<p><strong>Hibernation process</strong></p>
<p>When you put your computer to &#8216;sleep&#8217; there are actually several ways in which it can be performed by the operating  system one of those being the hibernation one. The hibernation process puts the contents of your memory into the hiberfil.sys file so that the state of all your running applications is preserved. By default when you enable hibernation the hiberfil.sys is created and filled with zeros. To enable hibernation you can run the following command in an elevated command shell:</p>
<pre>powercfg.exe -H on</pre>
<p>If you want to also control the size you can do:</p>
<pre>powercfg.exe -H -Size 100</pre>
<p>An interesting fact to note is that Windows 7 sets the size of the hibernation file size to 75% of your memory size by default. According to Microsoft documentation (2) this means that hibernation process could fail if it&#8217;s not able to compress the memory contents to fit in the hibernation file. This of course is useful information since it indicates that the contest of the hibernation file is compressed which usually will make basic analysis like &#8216;strings&#8217; pretty useless.</p>
<blockquote><p>if you use strings always go for &#8216;strings -a &lt;inputfile&gt;&#8217; read <a href="http://lcamtuf.blogspot.nl/2014/10/psa-dont-run-strings-on-untrusted-files.html" target="_blank">this</a> post if you are wondering why.</p></blockquote>
<p>The hibernation file usually resides in the root directory of the system drive, but it&#8217;s not fixed. If an administrators wants to change the location he can do so by editing the following registry key as explained by <a href="http://msdn.microsoft.com/en-us/library/ms912851(v=winembedded.5).aspx" target="_blank">this</a> (3) msdn article:</p>
<pre>Key Name: HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\Memory Management\
Value Name: PagingFiles
Type: REG_MULT_SZ
Data: C:\pagefile.sys 150 500
In the Data field, change the path and file name of the pagefile, along with the minimum and maximum file size values (in megabytes).</pre>
<p>So if you are performing an incident response or forensic investigation make sure you check this registry key before you draw any conclusion if the hiberfil.sys file is absent from it&#8217;s default location. Same goes for creating memory images using hibernation, make sure you get the 100% and write it to a location which doesn&#8217;t destroy evidence or where the evidence has already been collected.</p>
<p>Where does the slack space come from you might ask? That&#8217;s an interesting question since you would assume that each time the computer goes into hibernation mode it would create a new hiberfil.sys file, but it doesn&#8217;t. Instead it will overwrite the current file with the contents it wants to save. This is what causes slack space, since if the new data is smaller in size than the already available files the data at the end of the file will still be available even if it&#8217;s not referenced by the new headers written to the file.</p>
<p>From a forensic standpoint that&#8217;s pretty interesting since the unreferenced but available data might contain important information to help the investigation along. If you are working with tools that automatically import / parse or analyse the hiberfil.sys file you should check / ask / test how they handle slack space. In a best case scenario they will inform you about the slack space and try to recover the information, in a less ideal scenario they will inform you that there is slack space but it&#8217;s not able to handle the data and in the worst case scenario it will just silently ignore that data and tell you the hibernation file has been processed successfully.</p>
<p> <a href="http://diablohorn.com/2014/12/10/parsing-the-hiberfil-sys-searching-for-slack-space/#more-1046" class="more-link">Read the rest of this entry &raquo;</a></p>
</div>

<div class="posttitle">
	<h2 class="pagetitle"><a href="http://diablohorn.com/2014/07/26/writing-your-own-blind-sqli-script/" rel="bookmark" title="Permanent Link to Writing your own blind SQLi&nbsp;script">Writing your own blind SQLi&nbsp;script</a></h2>
	<small>
		Posted: July 26, 2014 in <a href="http://diablohorn.com/category/general/" rel="category tag">general</a>, <a href="http://diablohorn.com/category/security/" rel="category tag">security</a>		<br />
		Tags: <a href="http://diablohorn.com/tag/blind-injection/" rel="tag">blind injection</a>, <a href="http://diablohorn.com/tag/python/" rel="tag">python</a>, <a href="http://diablohorn.com/tag/sql-injection/" rel="tag">sql injection</a>, <a href="http://diablohorn.com/tag/sqli/" rel="tag">sqli</a>	</small>
</div>
<div class="postcomments"><a href="http://diablohorn.com/2014/07/26/writing-your-own-blind-sqli-script/#respond">0</a></div>
<div class="entry">
	<p>We all know that <a href="http://sqlmap.org/" target="_blank">sqlmap</a> is a really great tool which has a lot of options that you can tweak and adjust to exploit the SQLi vuln you just found (or that sqlmap found for you). On rare occasions however you might want to just have a small and simple script or you just want to learn how to do it yourself. So let&#8217;s see how you could write your own script to exploit a blind SQLi vulnerability. Just to make sure we are all on the same page, here is the blind SQLi definition from <a href="https://www.owasp.org/index.php/Blind_SQL_Injection" target="_blank">OWASP</a>:</p>
<blockquote><p>Blind SQL (Structured Query Language) injection is a type of <a title="SQL Injection" href="https://www.owasp.org/index.php/SQL_Injection" target="_blank">SQL Injection</a> attack that asks the database true or false questions and determines the answer based on the applications response.</p></blockquote>
<p>You can also roughly divide the exploiting techniques in two categories (like owasp does) namely:</p>
<ul>
<li>content based
<ul>
<li>The page output tells you if the query was successful or not</li>
</ul>
</li>
<li>time based
<ul>
<li>Based on a time delay you can determine if your query was successful or not</li>
</ul>
</li>
</ul>
<p>Of course you have dozens of variations on the above two techniques, I wrote about one such variation a <a href="http://diablohorn.wordpress.com/2011/07/08/efficient-but-slow-blind-sql-injection-data-extraction/" target="_blank">while ago</a>. For this script we are going to just focus on the basics of the mentioned techniques, if you are more interested in knowing how to find SQLi vulnerabilities you could read my article on <a href="http://diablohorn.wordpress.com/2014/02/09/solving-roguecoders-sqli-challenge/" target="_blank">Solving RogueCoder’s SQLi challenge</a>. Since we are only focusing on automating a blind sql injection, we will not be building functionality to find SQL injections.</p>
<p>Before we even think about sending SQL queries to the servers, let&#8217;s first setup the vulnerable environment and try to be a bit realistic about it. Normally this means that you at least have to login, keep your session and then inject. In some cases you might even have to take into account <a href="http://en.wikipedia.org/wiki/Cross-site_request_forgery" target="_blank">CSRF</a> tokens which depending on the implementation, means you have to parse some HTML before you can send the request. This will however be out of scope for this blog entry. If you want to know how you could parse HTML with python you could take a look at my <a href="http://diablohorn.wordpress.com/2012/11/22/credential-scavenger/" target="_blank">credential scavenger</a> entry.</p>
<p>If you just want the scripts you can find them in the <a href="https://github.com/DiabloHorn/example_bsqli_scripts" target="_blank">example_bsqli_scripts</a> repository on my github, since this is an entry on how you could write your own scripts all the values are hard coded in the script.</p>

	<h2 class="pagetitle"><a href="http://diablohorn.com/2014/07/13/physical-home-security-options/" rel="bookmark" title="Permanent Link to Physical home security&nbsp;options">Physical home security&nbsp;options</a></h2>
	<small>
		Posted: July 13, 2014 in <a href="http://diablohorn.com/category/memotoself/" rel="category tag">MemoToSelf</a>, <a href="http://diablohorn.com/category/security/" rel="category tag">security</a>		<br />
		Tags: <a href="http://diablohorn.com/tag/alarm/" rel="tag">alarm</a>, <a href="http://diablohorn.com/tag/burglarly/" rel="tag">burglarly</a>, <a href="http://diablohorn.com/tag/locks/" rel="tag">locks</a>, <a href="http://diablohorn.com/tag/physical-security/" rel="tag">physical security</a>	</small>
</div>
<div class="postcomments"><a href="http://diablohorn.com/2014/07/13/physical-home-security-options/#respond">0</a></div>
<div class="entry">
	<p>During one of those boring afternoons I noticed that most embassy buildings seem to have more or less the same amount of visible physical protection, this made me wonder what the available options are if you&#8217;d want to protect your own house. These are just some quick notes after searching around the internet on another boring afternoon, most of the options have an additional wide variety of configuration possibilities. So if you decide to implement any of the mentioned options, please do some research yourself, since these are just some starting points and you should choose the appropriate configuration yourself.</p>
<p>Keep in mind that (as far as I know) most options discussed here can be bypassed. Like with all security measures you should base the choices you make on a layered approach. The sum of all the protective measures should buy you enough time to detect a break in and react before any valuables are stolen. Feel free to leave additional measures in the comments or how to bypass the measures in this article. Do keep in mind that these measures will be a lot less effective if they decide to break in while you are at home, also these security measure are not aimed at preventing social engineering attacks. As usual I&#8217;ve got no clue what the rules in your country are, so read up on them before you implement some of the following options.</p>
<p>Oh and if you want an example of how physical security measures can by bypassed then you&#8217;ll probably enjoy this article. It&#8217;s about one of the biggest ($100 million) diamond heists in Belgium and how the attackers bypassed ten physical security measures: <a href="http://archive.wired.com/politics/law/magazine/17-04/ff_diamonds?currentPage=all" target="_blank">The Untold Story of the World&#8217;s Biggest Diamond Heist</a></p>
