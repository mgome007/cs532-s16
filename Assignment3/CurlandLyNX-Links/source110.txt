			<a itemprop="url" href="http://www.joshualong.co.uk/windows-10-forced-windows-updates/" title="Permalink to Windows 10: Forced Windows updates" rel="bookmark">Windows 10: Forced Windows updates</a>
		</h2>
							<div class="entry-meta"><span class="entry-date"><a href="http://www.joshualong.co.uk/windows-10-forced-windows-updates/" rel="bookmark"><time class="entry-date published" datetime="2015-08-24T12:06:50+00:00" itemprop="datepublished">24th August 2015</time></a></span> by <span class="author vcard" itemscope itemprop="author" itemtype="http://schema.org/Person"><a class="url fn n" href="http://www.joshualong.co.uk/author/joshua/" rel="author" itemprop="url"><span itemprop="name">Joshua</span></a></span><span class="sep">&middot;</span><span class="comments-link"><a href="http://www.joshualong.co.uk/windows-10-forced-windows-updates/#respond" itemprop="discussionUrl">0 Comments</a></span></div>
			</header>
								<div class="entry-content" itemprop="articleBody">
						<p>The forced updates are by far the trickiest aspect of Windows 10. Yet another problem that can occur is when a Windows update coincides with a third party driver management software update. This can cause a crash. The third party software can be halted, but the Windows update cannot. Your best bet is uninstall third party driver management and letting Windows take care of all the system updates, which gives Microsoft power over another company&#8217;s products, which isn&#8217;t cool. I hope there is a way to stop this happening in the near future.</p>

		</h2>
							<div class="entry-meta"><span class="entry-date"><a href="http://www.joshualong.co.uk/windows-7-import-from-network-to-windows-moviemaker/" rel="bookmark"><time class="entry-date published" datetime="2015-08-24T12:01:26+00:00" itemprop="datepublished">24th August 2015</time></a></span> by <span class="author vcard" itemscope itemprop="author" itemtype="http://schema.org/Person"><a class="url fn n" href="http://www.joshualong.co.uk/author/joshua/" rel="author" itemprop="url"><span itemprop="name">Joshua</span></a></span><span class="sep">&middot;</span><span class="comments-link"><a href="http://www.joshualong.co.uk/windows-7-import-from-network-to-windows-moviemaker/#respond" itemprop="discussionUrl">0 Comments</a></span></div>

							<div class="entry-content" itemprop="articleBody">
						<p>Ok, this is bizarre &#8211; Windows Live MovieMaker won&#8217;t let you import network files by default. Okay, so they&#8217;ll be large and may monopolise network traffic for a while, but shouldn&#8217;t that be your choice?</p>
<p>Anyway, a quick Registry tweak will solve this annoying problem. Simply run REGEDIT, browse to HKEY_CURRENT_USER\Software\Microsoft\Windows Live\Movie Maker, add a DWORD value called AllowNetworkFiles and set it to 1: job done.</p>

							<div class="entry-meta"><span class="entry-date"><a href="http://www.joshualong.co.uk/windows-7-standardising-the-taskbar/" rel="bookmark"><time class="entry-date published" datetime="2015-08-24T11:57:09+00:00" itemprop="datepublished">24th August 2015</time></a></span> by <span class="author vcard" itemscope itemprop="author" itemtype="http://schema.org/Person"><a class="url fn n" href="http://www.joshualong.co.uk/author/joshua/" rel="author" itemprop="url"><span itemprop="name">Joshua</span></a></span><span class="sep">&middot;</span><span class="comments-link"><a href="http://www.joshualong.co.uk/windows-7-standardising-the-taskbar/#respond" itemprop="discussionUrl">0 Comments</a></span></div>
			</header>
								<div class="entry-content" itemprop="articleBody">
						<p>We love the Windows 7 taskbar, but many people seem less than impressed with the new approach to taskbar buttons, finding it difficult to tell at a glance whether an icon is a running application or a pinned shortcut. If this sounds like you then there&#8217;s an easy way to restore more standard taskbar buttons, though &#8211; right-click the taskbar, select Properties, and set Taskbar Buttons to &#8220;Never combine&#8221; or &#8220;Combine when taskbar is full&#8221;.</p>
<p>You can even restore the old Quick Launch toolbar in just a few clicks. Simply right-click the taskbar, click Toolbars &gt; New Toolbar, type %userprofile%\AppData\Roaming\Microsoft\Internet Explorer\Quick Launch in the folder box and click Select Folder. The Quick Launch toolbar should then reappear, and you can move and resize it to suit your needs.</p>
