				<h2 class="entry-title"><a href="http://bachorz.eu/WoPr3/all-functionals-supporting-escf-in-bash-script/" title="Permalink to All functionals supporting escf in bash script" rel="bookmark">All functionals supporting escf in bash script</a></h2>
				<div class="entry-date"><abbr class="published" title="2014-02-02T23:48:19+0100">Sunday, February 2, 2014</abbr></div>
				<div class="entry-content">
<div class="pdfprnt-top-right"><a href="http://bachorz.eu/WoPr3/all-functionals-supporting-escf-in-bash-script/?page-id=342&#038;print=pdf" target="_blank"><img src="http://www.bachorz.eu/WoPr3/wp-content/plugins/pdf-print/images/pdf.png" alt="image_pdf" title="View PDF" /></a><a href="http://bachorz.eu/WoPr3/all-functionals-supporting-escf-in-bash-script/?page-id=342&#038;print=print" target="_blank"><img src="http://www.bachorz.eu/WoPr3/wp-content/plugins/pdf-print/images/print.gif" alt="image_print" title="Print Content" /></a></div>
<p>Simple, but sometimes can help. If you are going to prepare the Turbomole control<br />
files with all functionals supporting electronic excited state calculation &#8211; use sed tool in bash script.<br />
It is assumed that control.template file contains the <strong>pbe0</strong> functional. In the 5th line there is a submission to the queuing system realized by other scripts (TMOLE_submit and TMOLE_commands).</p>
<pre class="brush: bash; collapse: false; title: ; wrap-lines: false; notranslate" title="">
#!/bin/bash
for func in b3-lyp b3-lyp_Gaussian bh-lyp b-lyp pbe s-vwn s-vwn_Gaussian
do
sed 's/pbe0/'$func'/g' control.template &gt; control
./TMOLE_submit 6AC_$func TMOLE_commands
done
</pre>

<p>An installation of this tool is pretty straightforward can be done via mac port (just <em>port install hevea</em>). 			<div id="post-215" class="hentry p3 post publish author-admin category-science y2012 m12 d20 h12">
				<h2 class="entry-title"><a href="http://bachorz.eu/WoPr3/calculations-with-turbomole/" title="Permalink to Calculations with Turbomole 1" rel="bookmark">Calculations with Turbomole 1</a></h2>
				<div class="entry-date"><abbr class="published" title="2012-12-20T11:02:03+0100">Thursday, December 20, 2012</abbr></div>
				<div class="entry-content">
<div class="pdfprnt-top-right"><a href="http://bachorz.eu/WoPr3/calculations-with-turbomole/?page-id=215&#038;print=pdf" target="_blank"><img src="http://www.bachorz.eu/WoPr3/wp-content/plugins/pdf-print/images/pdf.png" alt="image_pdf" title="View PDF" /></a><a href="http://bachorz.eu/WoPr3/calculations-with-turbomole/?page-id=215&#038;print=print" target="_blank"><img src="http://www.bachorz.eu/WoPr3/wp-content/plugins/pdf-print/images/print.gif" alt="image_print" title="Print Content" /></a></div>
<p style="text-align: justify;">Below I want to show short guide related to the usage of  quantum chemistry package: Turbomole.</p>
<p style="text-align: justify;">First thing is to appropriately set up environmental variables. To do this just go to your Turbomole directory and type:</p>

<div class="wp-shkshell"><span class="wp-shkshell-prompt">$</span> <span class="wp-shkshell-command">source</span> Config_turbo_env<br /></div>

<p>Now you are ready to execute any of Turbomole programs/scripts. You can easily check it by typing:</p>

<div class="wp-shkshell"><span class="wp-shkshell-prompt">$</span> <span class="wp-shkshell-command">which</span> ridft<br />_some_path_/ridft<br /></div>

<p style="text-align: justify;">In the Turbomole package handling of input data is slightly different than in the case of many other quantum chemistry packages. Instead of one input file (like in the case of Gaussian) there are few other files that contain entire information about considered system. From the point of view of the user it is important to prepare the geometry of molecule, for example in the xyz format (using e.g. Molden program). The rest can be generated with the define program (a part of Turbomole package). However, Turbomole has its own format in which the geometry is expressed. To obtain this file you should use script x2t in the following way:</p>

<div class="wp-shkshell"><span class="wp-shkshell-prompt">$</span> <span class="wp-shkshell-command">x2t</span> geometry_in_the_xyz_format <span class="wp-shkshell-special">&</span>gt<span class="wp-shkshell-special">;</span> <span class="wp-shkshell-command">coord</span><br /></div>

<p style="text-align: justify;">Now your geometry is ready to go and define program should be used. In principle define is an interactive program that takes the user through couple of screens where one can load the geometry, set up the basis set and so on. It is a good idea to get familiar with define, but going through it routinely is a bit slow and a good idea is to write a short input to define. This input should contain all information which normally would be entered interactively. Below and exaple of such a input file:</p>

<div class="wp-shkshell"><span class="wp-shkshell-prompt">$</span> <span class="wp-shkshell-command">cat</span> define.inp<br />a coord<br />ired<br />*<br />b all def2-SV(P)<br />*<br />eht<br />0<br />ri<br />jbas<br />*<br />on<br />*<br />dft<br />on<br />*<br />*<br /></div>

<p>Starting from the beginnig we have the following commands:<br />
a coord &#8211; get the geometry from the coord file<br />
ired &#8211; impose the internal redundant coordinates (for geometry optimization)<br />
b all def2-SV(P) &#8211; choose the def2-SV(P) as the orbital basis set<br />
eht &#8211; compute the initial Huckel guess for the molecular orbitals<br />
0 &#8211; here we want to have the neutral molecule (charge=0)<br />
ri &#8211; use the resolution-of-the-identity for Coulomb integrals (calles sometimes the density fitting)<br />
jbas &#8211; take the auxiliary basis set which is adequate to orbital basis (with the same name, but this is completely different basis set)<br />
dft &#8211; turn on the dft mode (the SCF procedure will be calculating the DFT rather than Hartree-Fock energy)</p>
