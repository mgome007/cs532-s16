<p>I landed on <a href="https://github.com/sindresorhus/file-type/issues/22">this issue</a> made for expandingÂ <a href="https://github.com/sindresorhus/file-type">sindresorhus/file-type</a> with more compressed file types. Â <a href="https://github.com/sindresorhus/file-type">file-type</a>Â is a Node.js module made for apps needing to identify files by their actual binary signature,Â similar toÂ <a href="https://github.com/threatstack/libmagic">threatstack/libmagic</a>Â for the C language. Â It has Â a straightforward and conciseÂ structure consisting of a series of <code>if</code> statements and a byte-by-byte examination of the current file stream, typically only a few bytes. Here&#8217;s an example of how it checks a file if it&#8217;s a GIF:</p>
<pre class="EnlighterJSRAW" data-enlighter-language="js">if (buf[0] === 0x47 &amp;&amp; buf[1] === 0x49 &amp;&amp; buf[2] === 0x46) {
    return {
        ext: 'gif',
        mime: 'image/gif'
    };
}</pre>
<h4>Digging in&#8230;</h4>
<p><a href="http://www.crackmes.de/">I&#8217;ve messed with hex dumps and debugging binary files before</a>, and this simply involved cross-referencing the first few bytes of a file with already-known signatures.</p>
<p>The first Â task was to get the <a href="https://github.com/sindresorhus/compressed-extensions/blob/master/compressed-extensions.json">lists of file types</a> <a href="https://github.com/sindresorhus/archive-extensions/blob/master/archive-extensions.json">needing signatures</a> and grabbing the relevant data from sources around the net, mostly from theÂ <a href="http://www.garykessler.net/library/file_sigs.html">File Signatures Table, compiled by Gary Kessler</a>.</p>
<p style="text-align: justify;"><div class='content-column one_half'>I created a quick spreadsheet to keep things in order, too.</div><div class='content-column one_half last_column'><a href="http://devbrainhack.co/wp-content/uploads/2015/11/Screen-Shot-2015-11-08-at-5.32.45-PM.png"><img class="aligncenter size-large wp-image-127" src="http://devbrainhack.co/wp-content/uploads/2015/11/Screen-Shot-2015-11-08-at-5.32.45-PM-1024x507.png" alt="Screen Shot 2015-11-08 at 5.32.45 PM" width="660" height="327" /></a></div><div class='clear_column'></div>
<p>To help out on quick inspections, I created tiny utility to help at first, calling it fsig</p>
<pre class="EnlighterJSRAW" data-enlighter-language="shell">#!/bin/bash

clear
xxd $1 | head</pre>
<p>And for inspecting temporary directories of common file types, I used this bash one-liner:</p>
<pre class="EnlighterJSRAW" data-enlighter-language="null">clear; for n in *; do echo $n ; xxd $n | head ; done</pre>
<p>&nbsp;</p>
<p style="text-align: justify;"><div class='content-column one_half'>For extracting signatures cleanly, IÂ ran into <a href="http://ridiculousfish.com/hexfiend/">HexFiend</a>Â for OSX. Â It&#8217;s highly recommended since it displays where the cursor is by byte and allows for clean copy-paste.</div>
<div class='content-column one_half last_column'><a href="http://devbrainhack.co/wp-content/uploads/2015/11/Screen-Shot-2015-11-08-at-4.10.52-PM.png"><img class="aligncenter size-full wp-image-113" src="http://devbrainhack.co/wp-content/uploads/2015/11/Screen-Shot-2015-11-08-at-4.10.52-PM.png" alt="Screen Shot 2015-11-08 at 4.10.52 PM" width="806" height="886" /></a></div><div class='clear_column'></div>
<h4>Making it work</h4>
<p>A couple of problems I ran into was the issue of files with similar signatures. Â For example, <code>deb</code> files are nearly identical toÂ files made with the <code>compress</code> utility. Â That means that care was needed when ordering the logic.</p>

<h4>Picking a partner: values and experience aren&#8217;t all sugar and spice</h4>
<p>Picking a partner is a kind of baking. Â You get the ingredients together, you lay them out in front of you, then crack, crack, crack,Â mix, mix, mix, whisk them all in a bowl and stick it in an oven. Â Your project, theÂ business, is like that oven.</p>

<p>Except, <em>sometimes</em>, we as bakers are blind. Â We as bakers mislabel our ingredients, or ignore the labels altogether. <p>People are the same. Â You and I, we hold a few flavors for public consumption. Â How we behave, the attitudes we project, our go-to methods for conflict resolution and problem-solving: we keep these as a sort of feature-set called upon as long as there&#8217;s clean, un-pressured energy to utilize before our fight-or-flight centers kick in.</p>
<p>Take multiple people, add a potentially big payout, and now we&#8217;ve got some heat. Â You&#8217;re going to run into situations when what you thought was a Beef-Wellington ends up being just a starter-log wrapped in yolk-glazed paper-mache. Â Volatile, flammable.</p>

<p>So, big note-to-self: <span style="text-decoration: underline;">put the time in beforehand.</span></p>
<ol>
<li>&#8216;Date&#8217; all potentialÂ partners, and be aware of the typical warning signs (ex. mismatched values, gut feelings, personal history)</li>
<li>Make sure the vision is there. Â No vision is like a train off the rails: crazy.</li>
<li>Make sure there&#8217;s an end-game. Get confident about it, learn the possibilities and contingencies.</li>
<li>Have an understanding of how each partner resolves uncertainty and conflict. Â One person making decisions out of fear in a partnership is a symptom that real communication has broken down and planning is out the window. Â That&#8217;s a recipe for burnout and disaster.</li>
<li>Make sure negotiating financials isn&#8217;t akin to pulling teeth. Â Someone who continually tries to negotiate you at every step is a sign that you&#8217;re not going to be working together long.</li>
