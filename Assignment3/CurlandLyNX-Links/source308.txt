										<span class='seperator'>-</span>											<span class="date">December 27, 2015</span>
										</p>
				</div>
						
		</div>
							
		
		
	
			<div class="post-entry">
		
		<div class="post-entry-text">
		<p>Great documentary, my take aways below.</p>
<p><iframe width="960" height="540" src="https://www.youtube.com/embed/AYBRbCLI4zU?feature=oembed" frameborder="0" allowfullscreen></iframe></p>
<ul>
<li>Darwin spent 5 years at sea collecting samples before he started analyzing any of them. Then, back at his home in England, he began reviewing them. He understood the seasons of life at almost an expert level.</li>
<li>Darwin would perform &#8216;stream of consciousness&#8217; or &#8216;mental rioting&#8217;, as he called it, in which he&#8217;d write notebooks worth of thought until something solidifies.</li>
<li>Whales have teeth in their embryo, but they don&#8217;t exist in adult whales because of Darwinistic changes throughout the lives of multiple generations of whale.</li>
<li>Darwin realized that creatures change before he knew why they changed. Thus, allowing himself to have insight before comprehension.</li>
<li>The most brutal things in nature (aka the most extreme conflict like death of an animal from another) is actually the <strong>most creative </strong>possible thing for the gene pool. Fascinating.</li>
</ul>

							<h2><a href="http://dain.io/plotting-your-side-projects-performance-using-emacs-org-mode-and-python/">Tracking Your Side Project&#8217;s Performance Using Emacs, Org-Mode, and Python</a></h2>
			
							<div class="date-author">
					<p>
																					<span class="author">Dain</span>
										<span class='seperator'>-</span>											<span class="date">November 30, 2015</span>
										</p>
				</div>
						
		</div>
							
		
		
	
			<div class="post-entry">
		
		<div class="post-entry-text">
		<p>I&#8217;ve been experimenting with using <a href="https://www.gnu.org/software/emacs/" target="_blank">Emacs</a> <a href="http://orgmode.org/" target="_blank">Org-Mode </a><a href="http://orgmode.org/manual/Tables.html" target="_blank">tables</a> to track my <em><a href="http://searchcrm.techtarget.com/definition/key-performance-indicator" target="_blank">key performance indicators</a> </em>for my different side projects.</p>
<p>I like the idea of using emacs and org-mode for this type of data because it allows for quick data analysis via the programming language of your choice.</p>
<p>Alternatively, when using Google Spreadsheets you don&#8217;t have any integration with a programming language right at-hand.</p>
<p>Having a deep integration with a programming language on the data you collect, is invaluable. In this case, we are using the Emacs integration with <a href="http://orgmode.org/worg/org-contrib/babel/" target="_blank">babel&#8217;s</a> <em><a href="https://en.wikipedia.org/wiki/Literate_programming" target="_blank">literate coding</a></em> source code blocks.</p>
<p>This article is meant to be a very basic bare-bones intro to emacs org-mode, passing data from org-mode tables to souce-code, plotting, and rendering the result.</p>
<p>Due to the fact that we want to keep this approachable, we will use very basic data that produces very basic plots for this workflow tutorial.</p>
<p>Let&#8217;s dive in&#8230;</p>
<p><strong>Here&#8217;s our scenario</strong></p>
<p>We have a side project called <a href="http://starthere.fm" target="_blank">StartHereFM</a> (spoiler: true story). <a href="http://starthere.fm">StartHereFM</a> is a podcast network and the team needs to be able to see a range of different statistics on a daily basis.</p>
<p>Here&#8217;s a preview of the final result, let&#8217;s reverse engineer it.</p>
<p><a href="http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6.png"><img class="alignnone wp-image-642 size-full" src="http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6.png" alt="Screen-Shot-2015-11-30-at-1.42.36-AM" width="1196" height="656" srcset="http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6-300x165.png 300w, http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6-1024x562.png 1024w, http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6-1100x603.png 1100w, http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6-650x357.png 650w, http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6-440x241.png 440w, http://dain.io/wp-content/uploads/2015/11/Screen-Shot-2015-11-30-at-1.42.36-AM6.png 1196w" sizes="(max-width: 1196px) 100vw, 1196px" /></a></p>
<p><em>I&#8217;m assuming you know how to install emacs, install org-mode, open a website and read the basics of org-mode.</em></p>
<p>Let&#8217;s first create the org-mode file: I named mine &#8216;dashboard.org&#8217;.</p>
<p>The next thing we need is the data, which we store in <a href="http://orgmode.org/manual/Tables.html" target="_blank">org-mode tables</a>. Pick a few stats from your side project, and go ahead and replicate this table setup above. Enter your dates on the left, and columns with the data rows. FYI: You can automate this step if you want, but I won&#8217;t go into detail about it. At a high level you can use apps like <a href="https://ifttt.com/" target="_blank">IFTT</a> to send data to a backend service that you write that formats the data then serializes it into your file&#8217;s table on Dropbox/etc.</p>
<p><em>NOTE: Make sure you use the &#8220;#+TBLNAME&#8221; syntax to put a name on the table, for use in code in a minute.</em></p>
<p>Now that we have the data, let&#8217;s go ahead and send it on over into a code block, in this case we are using <a href="https://www.python.org/download/releases/3.0/">Python 3</a> primarily to take advantage of the <a href="http://www.scipy.org/" target="_blank">SciPy</a> stack.</p>
<pre class="brush: python; title: ; notranslate" title="">
** StartHereFM

     import numpy as np
     import matplotlib.pyplot as plt
     import matplotlib.dates as mdates
     import datetime 
</pre>
<p>You can see the source code block begins with<em> &#8220;#+begin_src&#8221;</em> and ends with <em>&#8220;#+end_src&#8221;</em>. Inside of that you put your code. You can hit <em>Ctrl-&#8216; </em>to edit the code in the block with syntax highlighting for that language. Then hit <em>Ctrl-&#8216;</em> again to get back to your source code block.</p>
<p>The next thing you&#8217;ll notice is the params I&#8217;m passing into the block. You can see I&#8217;m setting a <em>&#8220;data&#8221;</em> variable equal to the table name I defined above. This is very easy to do with org-mode and is a very cool way to toss data around. Thanks org! I&#8217;m also passing in <a href="http://orgmode.org/manual/Code-block-specific-header-arguments.html" target="_blank">a few options</a> to verify the org file displays the <em>stdout </em>of the script&#8217;s execution. And you can ignore the width bit.</p>
<p>Now that we have the data in the source code block with the SciPy stack, let&#8217;s make use of it!</p>
<p>First, let&#8217;s assign all columns data to a variable.</p>
<p>dates, daily_dls, total_dls, total_yt_views, site_views, email_list, twitter_followers = zip(*data)<br />
Next, let&#8217;s clean the data. Also called &#8220;data munging&#8221;.</p>
<p>Actually, pause, before we jump in let&#8217;s review data analysis 101 which states that there are four phases to managing data: <a href="https://en.wikipedia.org/wiki/Data_extraction" target="_blank">data extraction</a>, <a href="https://en.wikipedia.org/wiki/Data_wrangling" target="_blank">data munging</a> (cleaning), <a href="http://searchdatamanagement.techtarget.com/definition/data-governance" target="_blank">data governance</a> (controlling data quality), and <a href="https://en.wikipedia.org/wiki/Data_architecture" target="_blank">data architecture</a> (breaking silo&#8217;d data warehouses).</p>
<p>Okay, now that we have that out of the way let&#8217;s dive into munging or cleaning this data..</p>
<pre class="brush: python; title: ; notranslate" title=""> 
     dates = list(dates)
     dates = dates[1:]

<p>Very basic cleaning procedure for this example, just converting some strings to integers, and removing some empty strings from an array, etc.</p>
<p><strong>The end goal of this cleaning step is to verify you have two arrays in their <em>ready-to-</em><em>plot </em><em>data-type</em>, which for now means integers and date objects.</strong></p>
<p>To that end, let&#8217;s get our dates array ready&#8230;</p>
<pre class="brush: python; title: ; notranslate" title="">

<p><em>NOTE: I&#8217;m aware there are better ways to write this in Python, but I&#8217;m not a big Python coder and this works fine for my scripts like this one.</em></p>
<p>Now the data is fairly cleaned up and we have our 3 arrays (1 dates array, and 2 integer arrays to plot) let&#8217;s go ahead and create the figures using <a href="http://matplotlib.org/" target="_blank">matplotlib</a> and lay the data onto it in the most simple possible fashion.</p>
<p>The first thing you have to ask when plotting data is how many plots do I need? Could I plot all of this data on one plot?</p>
<p>Plotting all the arrays of data on one figure usually only works if the data sets are within the same range. <strong>If you have a diverse pool of ranges on the y axis the visualization will lose its ocular efficiency.</strong></p>
<p>Based on that, we know that this data is very different and probably won&#8217;t work on 1 figure. Why? Mainly because we are plotting <em>total_downloads </em>which is near the 100 thousands but the <em>daily_downloads </em>is very low around the couple hundred mark. Thus, this will be optimally plotted in two separate plots.</p>
<p>So, we are going to create two plots: the first is <em>total downloads</em> and the second is <em>daily downloads</em>. They both are going to have the dates in the x axis for this example, and the y axis will be the respective download count.</p>
<pre class="brush: python; title: ; notranslate" title="">
     fig = plt.gcf()
     fig.set_size_inches(5,2)

     fig2.set_size_inches(5,2)
     fig3 = plt.figure()
     fig3.set_size_inches(5,2)

     fig.gca().xaxis.set_major_formatter(mdates.DateFormatter('%b %d %Y'))
     fig.gca().xaxis.set_major_locator(mdates.DayLocator())

     fig2.gca().xaxis.set_major_formatter(mdates.DateFormatter('%b %d %Y'))
     fig2.gca().xaxis.set_major_locator(mdates.DayLocator())

     fig3.gca().xaxis.set_major_formatter(mdates.DateFormatter('%b %d %Y'))
     fig3.gca().xaxis.set_major_locator(mdates.DayLocator())

     ax = fig.add_subplot(111)
     ax2 = fig2.add_subplot(111)
     ax3 = fig3.add_subplot(111)

     ax.plot(_dates, _total_dls, label=&quot;Total Downloads&quot;)
     ax.legend(loc='upper center', bbox_to_anchor=(0.5, 1.09))
     fig.tight_layout()
     fig.savefig('starthere.png')

     ax2.plot(_dates, daily_dls, label=&quot;Daily Downloads&quot;)
     ax2.legend(loc='upper center', bbox_to_anchor=(0.5, 1.09))
     fig2.tight_layout()
     fig2.savefig('starthere-daily.png')

     ax3.plot(_dates, _total_yt_views, label=&quot;YouTube Views&quot;)
     ax3.legend(loc='upper center', bbox_to_anchor=(0.5, 1.09))
     fig3.tight_lay
     fig3.savefig('starthere-ytviews.png')

<p><em>NOTE: I&#8217;m aware this is not compact or DRY, but it works for mer. Also, you&#8217;ll notice we are plotting a third chart &#8211; please ignore that for this tutorial.</em></p>
<p>The first thing we do is create the figures and set their size. You can read up on the options by visiting the <a href="http://matplotlib.org/contents.html" target="_blank">matplotlib documentation</a>.</p>
<p>Next we set the formatting for the date axis on the plots. Then we add subplots. Then we call plot! <a href="http://matplotlib.org/api/pyplot_api.html" target="_blank">Plot simply takes</a> as params an <em>x, y, and label. </em>How amazing right?</p>
<p>I then call<em> tight_layout()</em>, which is a nice helper function that basically makes tiny charts look better by using a different spacing technique on the axis labels.</p>
<p><strong>Then I save the figure, because want to the visualization to appear in the document.</strong></p>
<p>How do we do that? Well, first you should know that Emacs is basically an <a href="http://unix.stackexchange.com/questions/986/what-are-the-pros-and-cons-of-vim-and-emacs" target="_blank">operating system for text</a>. Thus, <a href="http://orgmode.org/manual/External-links.html" target="_blank">files can be linked to and fro&#8217;</a>, and you do that by using the syntax: <em>&#8220;[[file_name.png]]&#8221;. </em></p>
<p>Here we want to put these links in for the plots we just saved.</p>
<pre class="brush: python; title: ; notranslate" title="">
   *StartHereFM Key Performance Indicators Over Time*
   [[starthere.png]]

   [[./starthere-daily.png]]
   
   [[./starthere-ytviews.png]]
</pre>
<p>Once these are set, you can put your cursor in the source code block and hit Ctrl-c Ctrl-c and it will run the <em>babel-execute </em>function for the language you listed in the <em>&#8220;#+begin_src python&#8221; </em>snippet.</p>
<p>Now your plots should be saved in the folder you&#8217;ve placed this <em>.org </em>file.</p>
<p><strong>The magic is now possible and you can </strong><strong>toggle between the <em>link to your plot</em> and <em>the plot image itself inline</em> with <em>Ctrl-c Ctrl-x Ctrl-v</em>.</strong></p>
<p>Here&#8217;s the <a href="https://gist.githubusercontent.com/dainmiller/9f2e38f7a09e25dcd2a9/raw/ca63ef5a0fa1dfe703ce16d90e04cfdb71e376ca/dash-example.org">full org file source</a> for you to run (<em>Ctrl-c Ctrl-c </em>in code block then <em>Ctrl-c Ctrl-x Ctrl-v</em>) in org-mode and learn from.</p>

<p>&nbsp;</p>
<p>That wraps the tutorial.</p>
<p>I&#8217;m a huge believer in the &#8220;you only change what you track, you only track what you measure&#8221; thing, and that&#8217;s why I think having good automated routines to plot your data for analysis later is hugely important.</p>
<p>What if these tables were being automatically populated and there was 100 columns of project statistics? At that point we are sort of replicating something like <a href="https://mixpanel.com/" target="_blank">Mixpanel</a> here, but that data is incredibly useful when planning features or prioritizing work as well. And because it&#8217;s integrated in your editor with a programming environment you can actually set up ways to ask questions of the data during stand up meetings. You can query your user data and basically &#8220;ask it&#8221; which feature should be prioritized over the other based on assumed impact and user benefit, etc.</p>
<p>Point is, <strong>it gets more interesting the more data you have</strong>. Always keep that in mind, and keep collecting.</p>
