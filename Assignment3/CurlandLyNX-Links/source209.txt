<h3 class='post-title entry-title'>
<a href='http://www.mad3linux.org/2015/11/recuperando-discos-rigidos-com-bad.html'>Recuperando Discos Rígidos com Bad Blocks ou Setores Defeituosos</a>
</h3>
<div class='post-header'>
<div class='post-header-line-1'><div class='post-share-buttons'>
<a class='goog-inline-block share-button sb-email' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=3041544565897369036&target=email' target='_blank' title='Enviar por e-mail'><span class='share-button-link-text'>Enviar por e-mail</span></a><a class='goog-inline-block share-button sb-blog' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=3041544565897369036&target=blog' onclick='window.open(this.href, "_blank", "height=270,width=475"); return false;' target='_blank' title='BlogThis!'><span class='share-button-link-text'>BlogThis!</span></a><a class='goog-inline-block share-button sb-twitter' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=3041544565897369036&target=twitter' target='_blank' title='Compartilhar no Twitter'><span class='share-button-link-text'>Compartilhar no Twitter</span></a><a class='goog-inline-block share-button sb-facebook' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=3041544565897369036&target=facebook' onclick='window.open(this.href, "_blank", "height=430,width=640"); return false;' target='_blank' title='Compartilhar no Facebook'><span class='share-button-link-text'>Compartilhar no Facebook</span></a><a class='goog-inline-block share-button sb-pinterest' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=3041544565897369036&target=pinterest' target='_blank' title='Compartilhar com o Pinterest'><span class='share-button-link-text'>Compartilhar com o Pinterest</span></a><div class='goog-inline-block google-plus-share-container'><g:plusone source='blogger:blog:plusone' href='http://www.mad3linux.org/2015/11/recuperando-discos-rigidos-com-bad.html' size='medium' width='300' annotation='inline'/></div>
</div>
</div>
</div>
<div class='post-body entry-content'>
Bad block ou setor defeituoso é o nome dado a uma área danificada em um disco rígido. É um problema físico, i. e., a mídia magnética do disco rígido está com defeito. Quando usamos um utilitário de disco como o Scandisk ou Norton Disk Doctor esses setores com defeito são marcados com um "B".<div>
<br /></div>
<div>
Vários usuários nos escrevem perguntando como proceder para recuperar discos rígidos com bad blocks. Muitos referem que os&nbsp;bad blocks&nbsp;desaparecem depois de formatar os discos rígidos em baixo nível.<br /><br />O que realmente acontece, porém, é que os atuais programas de formatação física não formatam fisicamente o disco. Se isso fosse possível, o disco rígido seria danificado, uma vez que as trilhas de um disco rígido tem um sinal chamado <i>servo</i> que funciona como um guia para a cabeça do disco rígido. Se realmente um disco rígido for&nbsp;formatado&nbsp;em baixo nível, esses servos seriam apagados e a cabeça do&nbsp;disco rígido seria incapaz de se mover.<br /><br />Programas de formatação de baixo nível são utilitários para detectar os setores defeituosos e para apagar o disco (por razões de segurança, por exemplo, após terminar um projeto confidencial), não efetuando - apesar do nome - a formatação de baixo nível.<br /><br />Estes programas têm uma função interessante, que é atualizar o mapa de setores defeituosos do disco. Quando você usa esta opção, o programa varre o disco procurando por setores defeituosos e atualiza o mapa do disco.<br /><br />Quando você faz uma formatação em alto nível (através do comando <i>format</i>), esse comando ignora os setores constantes nessa tabela de setores defeituosos. Assim, não haverá qualquer sector marcado com B ("Bad Block") no sistema de arquivo (por exemplo FAT, NTFS, ext4), embora os setores defeituosos permanecem no disco.<br /><br />Os setores defeituosos não são removidos, mas apenas marcados nessa tabela de setores defeituosos, resultando na evasão deles pelo sistema (em outras palavras, os setores se tornam ocultos).<br /><br />Se novos setores ruins continuarem a ocorrer depois de executar este procedimento, você deve se livrar do disco, pois a superfície magnética está se deteriorando, por algum motivo.<br /><br />O melhor programa para ser aplicada no processo é o do próprio fabricante. Entre no site do fabricante e baixe o utilitário específico para disco rígido.<div>
<br /></div>
<h3>
Alternativa no Linux</h3>
<div>
Existe um comando chamado <b>badblocks</b>&nbsp;que faz parte do projeto <a href="http://e2fsprogs.sourceforge.net/" target="_blank">e2fsprogs</a> (livre). Aprenda como usar através do manual:</div>
<blockquote class="tr_bq">
man badblocks</blockquote>
<div>

<h3 class='post-title entry-title'>
<a href='http://www.mad3linux.org/2015/10/usando-docker-em-seu-projeto-php.html'>Usando Docker em seu Projeto PHP</a>
</h3>
<div class='post-header'>
<div class='post-header-line-1'><div class='post-share-buttons'>
<a class='goog-inline-block share-button sb-email' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=1630599041622348592&target=email' target='_blank' title='Enviar por e-mail'><span class='share-button-link-text'>Enviar por e-mail</span></a><a class='goog-inline-block share-button sb-blog' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=1630599041622348592&target=blog' onclick='window.open(this.href, "_blank", "height=270,width=475"); return false;' target='_blank' title='BlogThis!'><span class='share-button-link-text'>BlogThis!</span></a><a class='goog-inline-block share-button sb-twitter' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=1630599041622348592&target=twitter' target='_blank' title='Compartilhar no Twitter'><span class='share-button-link-text'>Compartilhar no Twitter</span></a><a class='goog-inline-block share-button sb-facebook' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=1630599041622348592&target=facebook' onclick='window.open(this.href, "_blank", "height=430,width=640"); return false;' target='_blank' title='Compartilhar no Facebook'><span class='share-button-link-text'>Compartilhar no Facebook</span></a><a class='goog-inline-block share-button sb-pinterest' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=1630599041622348592&target=pinterest' target='_blank' title='Compartilhar com o Pinterest'><span class='share-button-link-text'>Compartilhar com o Pinterest</span></a><div class='goog-inline-block google-plus-share-container'><g:plusone source='blogger:blog:plusone' href='http://www.mad3linux.org/2015/10/usando-docker-em-seu-projeto-php.html' size='medium' width='300' annotation='inline'/></div>
</div>
</div>
</div>
<div class='post-body entry-content'>
<h1>
<a href="https://draft.blogger.com/null" id="Introduo_0"></a>Introdução</h1>
Depois de muito tempo tentando apenas entender o que é o Docker, achei melhor aprender colocando a mão na massa: Fazer um projeto que usa PHP e PostgreSQL rodar no Docker.<br />
O projeto escolhido foi o <a href="http://comsolid.github.io/sige/">SiGE</a>, um Sistema de Gerência de Eventos utilizado principalmente na organização do <a href="http://comsolid.org/">COMSOLiD</a> (evento realizado por alunos e ex-alunos do IFCE Campus Maracanaú).<br />
<br />
<h3>
O que é o Docker?</h3>
<h3>
&nbsp;</h3>
<div class="separator" style="clear: both; text-align: center;">
<a href="http://3.bp.blogspot.com/-ixnbUywtcyI/VhKzOzVvALI/AAAAAAAACKQ/pqYhA7VhWIw/s1600/DockerLogo.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="95" src="http://3.bp.blogspot.com/-ixnbUywtcyI/VhKzOzVvALI/AAAAAAAACKQ/pqYhA7VhWIw/s400/DockerLogo.png" width="400" /></a></div>
<h3>
</h3>
<br />
Segundo o site<br />
<blockquote>
Docker é uma plataforma de código aberto para construção, implantação e execução de aplicações distribuídas. Ele oferece a programadores, equipes de desenvolvimento e engenheiros um conjunto de ferramentas comum para tirar vantagem da natureza distribuída e em rede de aplicações modernas.</blockquote>
Ou seja, ao montar um ambiente de desenvolvimento fica mais fácil imitar o ambiente de produção e testes. Imagine o seguinte cenário:<br />
<br />
O servidor de produção possui o PHP 5.4 instalado juntamente com o PostgreSQL 9.3. Você, desenvolvedor, tem o PHP 5.5 e o PostgreSQL 9.4. Ao realizar uma tarefa você percebe que o código usado só funciona na versão 5.5 do PHP e que na 5.4, versão do servidor, não existe essa funcionalidade ainda. O que acontece é que você só vai perceber o erro quando o servidor estiver em produção, o que com certeza irá causar uma dor de cabeça.<br />
<br />
Usando Docker você é capaz de usar exatamente as mesmas versões das ferramentas, tanto no servidor quanto local, sem que elas causem conflitos em sua máquina. O Docker utiliza uma tecnologia chamada <a href="https://linuxcontainers.org/lxc/introduction/">LXC</a> ou Linux Containers (na verdade essa é a tecnologia padrão, mas existem outras).<br />
<br />
Podemos pensar no LXC como uma máquina virtual leve, isso porque diferente da máquina virtual que sobe uma instância do sistema, o LXC usa o mesmo kernel da máquina host com a vantagem do Kernel namespaces - Componentes virtuais: ipc, uts, mount, pid, network e user.<br />
<br />
<div class="separator" style="clear: both; text-align: center;">
<a href="http://4.bp.blogspot.com/-LN9cdLP5JxE/VhKzqashjZI/AAAAAAAACKY/2l7rVpcPoIk/s1600/docker-vm-container.png" imageanchor="1" style="margin-left: 1em; margin-right: 1em;"><img border="0" height="360" src="http://4.bp.blogspot.com/-LN9cdLP5JxE/VhKzqashjZI/AAAAAAAACKY/2l7rVpcPoIk/s640/docker-vm-container.png" width="640" /></a></div>
<br />
<h3>
Instalação</h3>
<h3>
&nbsp;</h3>
<h3>
</h3>
No Linux: <a href="http://docs.docker.com/linux/step_one/">http://docs.docker.com/linux/step_one/</a><br />
Além do Docker é ideal instalar também o Docker Compose <a href="http://docs.docker.com/compose/install/">http://docs.docker.com/compose/install/</a><br />
<br />
<h3>
Utilização</h3>
<h3>
&nbsp;</h3>
<h4>
<a href="https://draft.blogger.com/null" id="Obervao_Abaixo_esto_apenas_os_passos_importantes_para_entender_a_ferramenta_veja_o_cdigo_completo_no_cdigofonte_26"></a>Obervação: Abaixo estão apenas os passos importantes para entender a ferramenta, veja o código completo no código-fonte.</h4>
As configurações de uma imagem do Docker devem ser feitas em um arquivo de texto, por padrão chamado de <code>Dockerfile</code>.<br />
O SiGE precisa inicialmente do PHP. Optei por utilizar a imagem <code>nmcteam/php56</code>, ou seja, PHP 5.6.<br />
Para utilizar a imagem basta colocar no seu <code>Dockerfile</code>:<br />
<br />
<pre><code><span class="hljs-built_in">FROM</span> nmcteam/php5<span class="hljs-number">6
</span>
COPY . /usr/share/nginx/html/site

<span class="hljs-built_in">WORKDIR</span> <span class="bash">/usr/share/nginx/html/site
</span>
<span class="hljs-built_in">RUN</span> <span class="bash">php -r <span class="hljs-string">"readfile('https://getcomposer.org/installer');"</span> | php
</span><span class="hljs-built_in">RUN</span> <span class="bash">[<span class="hljs-string">"/bin/bash"</span>, <span class="hljs-string">"-c"</span>, <span class="hljs-string">"php composer.phar install"</span>]</span></code></pre>
<pre><code><span class="bash">&nbsp;</span></code></pre>
O arquivo acima mostra que nossa imagem vai ser criada a partir de <code>nmcteam/php56</code> (FROM). Tudo que estiver no diretório atual (<code>.</code>) deve ser colocado em <code>/usr/share/nginx/html/site</code> (COPY). Ao executar comandos, execute a partir de <code>/usr/share/nginx/html/site</code> (WORKDIR). Em seguida o SIGE utiliza o <a href="https://getcomposer.org/">composer do PHP</a> (<b>Não confundir com o Docker Compose!</b>), para isso é preciso instalá-lo (RUN).<br />
No final das contas esse arquivo é como um script bash que executa alguns comandos para configurar o sistema.<br />
Para fazer o build da imagem execute:<br />
<br />
<pre><code><span class="hljs-label">docker</span> <span class="hljs-keyword">build </span>-t main/sige .</code></pre>
<pre><code>&nbsp;</code></pre>
A opção <code>-t</code> é o nome da sua imagem, dê o nome adequado.<br />
Arquivo completo com outros comandos necessários somente para o SiGE:
<a href="https://github.com/comsolid/sige/blob/master/Dockerfile">https://github.com/comsolid/sige/blob/master/Dockerfile</a><br />
Em seguida criei um <code>Dockerfile</code> separado para o PostgreSQL. Quando usar o Docker tente sempre separar os componentes do sistema em pequenos serviços que funcionam independentes um do outro (Micro Services!).<br />
<br />
<pre><code><span class="hljs-built_in">FROM</span> postgres:<span class="hljs-number">9.4</span>.<span class="hljs-number">4</span>

<span class="hljs-built_in">COPY</span> <span class="bash">docker-entrypoint-initdb.d/database.sql /docker-entrypoint-initdb.d/database.sql
</span>
<span class="hljs-built_in">RUN</span> <span class="bash">pg_createcluster <span class="hljs-number">9.4</span> main</span></code></pre>
<pre><code><span class="bash">&nbsp;</span></code></pre>
A equipe do PostgreSQL oferece uma imagem oficial. Ao subir uma instância dessa imagem (container), um script irá executar qualquer arquivo <code>.sql</code> dentro do diretório <code>docker-entrypoint-initdb.d/</code>, use quando precisar subir uma base de dados populada (COPY).<br />
Usaremos também a imagem do Nginx, mas como ela não precisa de muita configuração usaremos apenas declarada no Docker Compose.<br />
<br />
<h3>
<a href="https://draft.blogger.com/null" id="Arquivo_dockercomposeyml_74"></a>Arquivo docker-compose.yml</h3>
<h3>
&nbsp;</h3>
O Docker Compose utiliza uma linguagem chamada <a href="http://yaml.org/">YML</a> para seu arquivo docker-compose.yml (arquivo padrão do Docker Compose).<br />
É uma linguagem simples como JSON. Abaixo um exemplo para iniciar um serviço Nginx:<br />
<br />
<pre><code><span class="hljs-attribute">web</span>:
  <span class="hljs-attribute">image</span>: nginx
  <span class="hljs-attribute">ports</span>:
    - <span class="hljs-string">"8080:8080"</span></code></pre>
<pre><code><span class="hljs-string">&nbsp;</span>
</code></pre>
Iniciamos com uma label (<code>web</code>), a partir da imagem <code>nginx</code>, e mapeando a porta <code>8080</code> para <code>8080</code> (a primeira indica a porta que você vai utilizar, a segunda é a porta interna, usada pelo container).<br />
Em seguida podemos colocar nossa imagem do PHP 5.6 e PostgreSQL 9.4 fazendo uso de <code>links</code>, que faz com que um container converse com outro:<br />
<br />
<pre><code><span class="hljs-attribute">web</span>:
  <span class="hljs-attribute">image</span>: nginx
  <span class="hljs-attribute">ports</span>:
    - <span class="hljs-string">"8080:8080"</span>
  <span class="hljs-attribute">links</span>:
    - php

<span class="hljs-attribute">php</span>:
  <span class="hljs-attribute">image</span>: main/sige
  <span class="hljs-attribute">links</span>:
    - pg

<span class="hljs-attribute">pg</span>:
  <span class="hljs-attribute">image</span>: sige/pg</code></pre>
<pre><code>&nbsp;</code></pre>
Veja que fizemos links de um serviço com outro para que eles conversem, ou seja, um serviço Nginx não conhece a linguagem PHP então ele pede que o serviço PHP execute o código para ele e entrega de volta o resultado.<br />
Da mesmo forma quando um trecho de código PHP tenta acessar o banco de dados do serviço PostgreSQL.<br />
Para subir todos os serviços de uma vez basta executar:<br />
<br />
<pre><code>docker-compose up <span class="hljs-operator">-d</span></code></pre>
<pre><code><span class="hljs-operator">&nbsp;</span>
</code></pre>
Como foi dito anteriormente o Docker utiliza LXC. Se você executar <code>ifconfig</code> em seu terminal vai ver várias interfaces de rede configuradas. É assim que os containers se comunicam. Para saber o IP principal basta procurar pela inteface <code>docker0</code>.<br />
Para ver os serviços rodando basta acessar (no meu caso): 172.17.42.1:8080<br />
<br />
<h3>
Conclusão</h3>
<h3>
&nbsp;</h3>
Lembrando que os trechos acima estão incompletos mas são essenciais.
Se eu fosse mostrar passo a passo você provavelmente ia cansar de ler e desistir. Então se você leu e se interessou abaixo estão os links de todos os arquivo envolvidos:<br />

</ul>
Você provavelmente não precisará mexer muita coisa, mas por exemplo meu arquivo <a href="https://github.com/comsolid/sige/blob/master/docker/nginx/vhost.conf">https://github.com/comsolid/sige/blob/master/docker/nginx/vhost.conf</a> está configurado para uso do Zend 1.12 (que usa reescrita (rewrite) da URI). Basta lembrar que a tecnologia não mudou, apenas aumentamos um nível de acesso. Quer dizer, se você tem um arquivo <code>vhost.conf</code> que roda fora do Docker, ele pode ser usado dentro do Docker sem problemas.<br />
É isso aí, até a próxima!

<h3 class='post-title entry-title'>
<a href='http://www.mad3linux.org/2015/06/convertendo-mkv-para-mp4-sem-perda.html'>Convertendo MKV para MP4 sem perda (lossless) e sem reencodar</a>
</h3>
<div class='post-header'>
<div class='post-header-line-1'><div class='post-share-buttons'>
<a class='goog-inline-block share-button sb-email' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=8231724474240041184&target=email' target='_blank' title='Enviar por e-mail'><span class='share-button-link-text'>Enviar por e-mail</span></a><a class='goog-inline-block share-button sb-blog' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=8231724474240041184&target=blog' onclick='window.open(this.href, "_blank", "height=270,width=475"); return false;' target='_blank' title='BlogThis!'><span class='share-button-link-text'>BlogThis!</span></a><a class='goog-inline-block share-button sb-twitter' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=8231724474240041184&target=twitter' target='_blank' title='Compartilhar no Twitter'><span class='share-button-link-text'>Compartilhar no Twitter</span></a><a class='goog-inline-block share-button sb-facebook' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=8231724474240041184&target=facebook' onclick='window.open(this.href, "_blank", "height=430,width=640"); return false;' target='_blank' title='Compartilhar no Facebook'><span class='share-button-link-text'>Compartilhar no Facebook</span></a><a class='goog-inline-block share-button sb-pinterest' href='https://www.blogger.com/share-post.g?blogID=4315013234587932472&postID=8231724474240041184&target=pinterest' target='_blank' title='Compartilhar com o Pinterest'><span class='share-button-link-text'>Compartilhar com o Pinterest</span></a><div class='goog-inline-block google-plus-share-container'><g:plusone source='blogger:blog:plusone' href='http://www.mad3linux.org/2015/06/convertendo-mkv-para-mp4-sem-perda.html' size='medium' width='300' annotation='inline'/></div>
</div>
</div>
</div>
<div class='post-body entry-content'>
Fazer conversão de vídeo demora muito e é um saco esperar. Atualmente, os dispositivos mais recentes (TVs, home teathers, consoles, celulares, tablets) conseguem reproduzir vários formatos e vários <i>codecs</i>. Entretanto, os modelos mais antigos ainda sofrem com a restrição de formatos e codecs. Por exemplo, quando você baixa um vídeo num formato (também chamado de contêiner) MKV ou MP4 e sua TV só reproduz um desses formatos, saiba que não é preciso 'reencodar' todo o vídeo e áudio. Basta fazer o chamado <i>demux</i> e <i>mux</i>, que é muito mais rápido, pois não realiza cálculos complexos para encodar (ou compactar) vídeo e áudio. Para fazer isso, você pode usar tanto o <a href="https://www.ffmpeg.org/" target="_blank">FFmpeg</a> quanto o&nbsp;<a href="https://libav.org/" target="_blank">Libav</a>.<br />
<br />
Como alguns releases do Ubuntu não têm o ffmpeg (<a href="http://blog.pkh.me/p/13-the-ffmpeg-libav-situation.html" target="_blank">explicação detalhada em inglês</a>&nbsp;[<a href="https://drive.google.com/file/d/0BwIZj8djOAj2QUhoeWZfVzZVdlE/view?usp=sharing" target="_blank">pdf</a>]), você terá que usar o <a href="apt://libav-tools" target="_blank">avconv</a>&nbsp;(pacote <a href="http://packages.ubuntu.com/trusty/libav-tools" target="_blank">libav-tools</a>, do projeto <a href="https://libav.org/" target="_blank">Libav</a>) que funciona basicamente com os mesmos parâmetros.<br />
<br />
<h3>
De MKV para MP4:</h3>
<blockquote class="tr_bq">
<span style="font-family: 'Courier New', Courier, monospace;">$ avconv -i input.mkv -c:v copy -c:a copy -sn output.mp4</span></blockquote>
<h3>
De MP4 para MKV:</h3>
<blockquote class="tr_bq">
<span style="font-family: 'Courier New', Courier, monospace;">$ avconv -i input.mp4 -c:v copy -c:a copy -sn output.mkv</span></blockquote>
